## 🌟 为什么存在？(Our Principles)

技术是崇高的，但学习者需要自律。我们以组织的形式存在，是为了建立一套**良性竞争**与**价值保护**的机制。

### 1. 核心驱动：热爱与收益
* **源于热爱**：加入组织的前提是热爱编程。无论是为了解决 Bug 还是创造工具，热情是第一生产力。
* **价值共赢**：我们不排斥利益。相反，我们鼓励通过开源提升个人品牌、获取人脉甚至商业机会。**让开源成为你简历上最亮的一笔。**

### 2. 行为准则：底线与素养
* **自我保护**：学会使用 **License** 保护自己的原创作品。这是对自己劳动的尊重，也是对未来的负责。
* **尊重版权**：严禁未经允许的抄袭、洗稿。学习他人代码时，**必须保留原作者署名**，并在心中（及文档中）致谢。
* **平视交流**：不诋毁、不轻视。只要肯努力，曾经的小白也会成为大牛。在技术讨论中，请保持谦虚与建设性。

### 3. 成长思维：长期主义
* **拒绝速成**：你的努力决定了你的高度。不要迷信捷径，代码行数不会骗人。
* **持续输出**：多年后回望，你会发现正是今天提交的每一个 Commit，构成了你职业生涯的基石。

加入组织后，我们会根据你的技术栈、活跃度及代码质量，将你邀请至相应的 **Team**。分组是为了提供更精准的协作环境，而非阶级划分。

## ♻️ 项目生态：转入与转出 (Project Lifecycle)

我们建立了一套**无损**的项目流转机制，旨在放大你的项目价值。

### 📥 将项目转入组织 (Inbound)
> **为什么要转入？** 获得组织背书、更高的曝光率、社区成员的协助与 PR。

**⚠️ 重要提示（请仔细阅读）：**
1.  **权限交接**：转入初期，你可能会暂时失去 `Admin` 权限（GitHub 机制），但通常会在 **12小时内** 由管理员重新赋予你。
2.  **预约机制**：建议提前联系管理员（Issue 或 邮件），预约在线时间，实现“秒转秒还权限”。
3.  **所有权**：你永远拥有项目的所有权。

<details>
<summary>👉 点击查看：项目转入详细步骤 (图文教程)</summary>

1.  打开你要转入的项目，点击 **Settings** 选项卡。
2.  滑到最下方 Danger Zone，点击 **Transfer**。
3.  在输入框中输入：
    * 第一行：`你的项目名`
    * 第二行：`qianqianup` (组织名)
4.  点击确认。我们会尽快处理并归还管理权。

</details>

<br/>

### 📤 将项目转出组织 (Outbound)
> **为什么要转出？** 项目需独立商业化、个人回退或其他原因。我们支持**随时、无损**转出。

**✨ 你不会失去什么：** Stars 和 Forks 都会保留，不会丢失。

<details>
<summary>👉 点击查看：项目转出详细步骤</summary>

1.  打开项目 **Settings**。
2.  滑到最下方点击 **Transfer**。
3.  在输入框中输入：
    * 第一行：`项目名`
    * 第二行：`你的个人用户名`
4.  点击确认。项目即刻回到你的个人账号下。

</details>
