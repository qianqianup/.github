### 📚 组织纲领 (Guidelines)

我们将辩证唯物主义的方法论应用于开源协作，制定以下行动准则：

#### 1. 实践论：认识与行动的辩证统一 🛠️
* **要想知道梨子的滋味，就要亲口尝一尝**：编程没有捷径。加入组织的首要任务是**动手**。无论理论多么完美，不经过代码实现的检验都是空中楼阁。
* **从感性认识到理性认识**：我们鼓励从模仿开始，但不能止步于模仿。要在不断的 Debug 和重构中，透过现象看本质，掌握技术的底层逻辑。

#### 2. 矛盾论：个人与集体的对立统一 ⚖️
* **正确处理个人利益与集体利益的关系**：开源不是“打土豪”，也不是单纯的索取。我们反对将公共资源私有化（非法牟利），主张通过贡献集体来实现个人价值的提升。
* **主要矛盾与次要矛盾**：在开发中，要分清主次。当版权（License）和原创性受到挑战时，这是原则问题，必须严肃对待；而在技术选型上的分歧，属于人民内部矛盾，可以求同存异。

#### 3. 团结—批评—团结：社区建设路线 🤝
* **虚心使人进步，骄傲使人落后**：技术领域没有永远的权威。无论你处于哪个阶段，都要保持谦虚谨慎的作风。切忌在 Issue 中搞一言堂，切忌轻视新手。
* **惩前毖后，治病救人**：对待 Bug 和代码缺陷，我们要像医生对待病人一样。指出问题的目的是为了解决问题（修复 Bug），而不是为了消灭同志（人身攻击）。
* **公式**：从团结的愿望出发，经过批评或者斗争，在新的基础上达到新的团结。

#### 4. 论持久战：量变引起质变 📈
* **前途是光明的，道路是曲折的**：技术成长不是直线式的，而是波浪式前进的。不要因为一时的编译失败或理解困难而气馁，这是事物发展的必经阶段。
* **反对速胜论，坚持持久战**：不积跬步，无以至千里。我们要摒弃浮躁的“速成”心态，坚持长期的代码积累。只有量的积累达到一定程度，才能实现技术水平质的飞跃。
